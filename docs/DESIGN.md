# tiny-pdf-viewer 設計書

ブラウザで PDF を表示する、ビルド不要・単一ページの軽量ビューア。
大きい PDF でも快適に開けることを主眼に置く。

このドキュメントは Claude Code で実装を引き継ぐための設計書である。

## 1. 目的とスコープ

- 同一サーバ上の PDF を nginx で静的配信し、ブラウザで閲覧する。
- PDF のファイルサイズが大きいため、**全体をダウンロードしてから描画しない**こと（HTTP Range Request による必要ページのみの取得）を前提とする。
- フレームワーク・ビルドツールを使わず、素の HTML / CSS / ES Modules で構成する。デプロイは「ファイルを置くだけ」。

### やること

- URL パラメタ（`file_url` / `page` / `binding` / `spread`）でビューアを起動。
- 見開き表示（デフォルト右綴じ）。
- ページ移動スライダー。
- 全画面表示。

### やらないこと（YAGNI）

- テキスト検索、サムネイル一覧、ズーム倍率 UI、目次（アウトライン）、読書位置記憶、注釈、本棚 / ライブラリ画面、認証。
  - これらは将来の拡張候補だが、いずれも本ビューアの素の JS の延長で後から足せる（または別アプリ化すべき規模）ため、初期スコープには含めない。

## 2. 大きい PDF への対応方針（前提知識）

- PDF は末尾に索引（xref / trailer）を持ち、各ページのオブジェクトがファイル内の byte 位置に散在する。
- PDF.js は「末尾の索引 → 目的ページのオブジェクトの byte 範囲」だけを Range Request で取得するため、**途中ページを開いても先頭から読む必要はない**。
- nginx は静的ファイル配信で Range Request（`Accept-Ranges: bytes`、`206 Partial Content`）をデフォルトで返すため、追加設定はほぼ不要。
- 配信時の注意:
  - **`application/pdf` に gzip をかけない**（Range が壊れる）。
  - 任意で **PDF を linearize**（`qpdf --linearize`）しておくと初回表示と堅牢性が向上する（必須ではない）。

## 3. 技術構成

- 言語 / 実行形態: 素の HTML / CSS / JavaScript（ES Modules）。ビルドなし。
- PDF 描画: [PDF.js](https://github.com/mozilla/pdf.js)（`pdfjs-dist`）。prebuilt から `pdf.min.mjs` と `pdf.worker.min.mjs` を `vendor/pdfjs/` に同梱する。
- 配信: nginx で静的配信。
- バージョン: 同梱した pdfjs-dist のバージョンを README に明記する（更新時は vendor を入れ替えるだけ）。

### モジュール責務

各モジュールは 1 つの役割を持ち、明確なインタフェースで連携する。

- **`navigation.js`（純ロジック・DOM非依存・テスト対象の中心）**
  見開きロジックの中核。状態（現在位置・モード・綴じ方向・総ページ数）から「表示すべきページ番号」「次 / 前」「スライダー値との対応」を計算する。DOM や PDF.js に一切依存しないため単体テストが容易。
- **`pdf-core.js`（PDF.js の隠蔽）**
  `pdfjs-dist` を内部に閉じ込める薄いラッパ。ドキュメントのロード、総ページ数の取得、指定ページの canvas への描画を提供する。PDF.js の API 変更の影響をここに局所化する。
- **`viewer.js`（配線）**
  URL パラメタの解釈、UI（スライダー・ボタン・トグル）の構築、全画面、リサイズ監視を担当し、`navigation` と `pdf-core` を繋ぐ。

この分割により、将来ズーム・検索・サムネイルを足す場合や、フレームワークへ移す場合でも `navigation` / `pdf-core` のコアを流用できる。

## 5. URL パラメタ

| パラメタ | 必須 | 既定 | 意味 |
|---|---|---|---|
| `file_url` | ✅ | — | 表示する PDF の URL |
| `page` | — | `1` | 初期表示ページ（その番号を含む見開きを表示） |
| `binding` | — | `right` | 綴じ方向。`right`（右綴じ＝右が先のページ） / `left`（左綴じ）。内部表現は rtl/ltr |
| `spread` | — | `on` | 見開き。`on` / `off`（単ページ） |

例: `index.html?file_url=/pdfs/novel.pdf&page=10`

## 6. 表示・操作仕様

### 6.1 見開き（spread）

- canvas を 2 枚横並びにして 2 ページを同時表示する。
- **ページのペアリング**: 表紙の特別扱いはしない。ページは `1-2 / 3-4 / 5-6 …` で固定的にペアを組む。
  - ページ番号 `N` が属するペアは `{2k+1, 2k+2}`（`k = floor((N-1)/2)`）。
- **綴じ方向による並び**（画面の左→右の配置）:
  - `ltr`（左綴じ）: 左 = 奇数ページ `2k+1`、右 = 偶数ページ `2k+2`。
  - `rtl`（右綴じ・既定）: 左 = 偶数ページ `2k+2`、右 = 奇数ページ `2k+1`（右が先に読むページ）。
- **`page` の解釈**: 指定ページを含む見開きを表示する（例: `page=10` → ペア `9-10` を表示）。
- 末尾がペアにならない場合（総ページが奇数で最後のページが単独）は、片側 1 枚のみ描画する。

### 6.2 単ページ（spread=off / フォールバック）

- canvas 1 枚で 1 ページを表示する。
- 後述の狭い画面フォールバックでも、この単ページ描画を用いる。

### 6.3 スライダー

- `<input type="range">` でページを移動する。
- 見開き時はペア単位（実質 2 ページ刻み）で移動する。スライダー値 ↔ 表示ページの対応は `navigation.js` が担当する。
- 単ページ時は 1 ページ刻み。

### 6.4 全画面

- 表示領域コンテナに対して `requestFullscreen()` を呼ぶトグルボタンを置く。
- 全画面状態の変化（`fullscreenchange`）に追従してフィットを再計算する。

### 6.5 フィット

- 既定は幅フィット。見開き時は 2 ページ並べた合計幅をビューポート幅に合わせる。
- 高 DPI 描画のため `devicePixelRatio` を考慮して canvas を描画する。

### 6.6 狭い画面の自動フォールバック

- リサイズを監視し、横幅が閾値未満（見開きにすると 1 ページが過度に小さくなる場合）には、`spread=on` でも単ページ描画に切り替える。
- ユーザの見開き設定（状態）は保持し、画面が広がれば見開きへ戻る。

### 6.7 表示モード切替トグル

- 見開き ⇄ 単ページを切り替える UI トグルを置く（`spread` パラメタの初期値を実行時に上書きできる）。

## 7. データフロー

1. `viewer.js` が URL パラメタを読み取る。
2. `pdf-core.load(file_url)` で PDF をロードし、総ページ数を取得する。
3. 総ページ数とパラメタ（`page` / `dir` / `spread`）を `navigation` に渡し、初期表示ページを算出する。
4. `pdf-core.renderPage()` で対象ページを canvas に描画する。
5. スライダー / ボタン / トグル / リサイズの操作で `navigation` が次の表示状態を決定し、再描画する。

## 8. エラー処理

- `file_url` が無い → 画面にメッセージを表示する。
- ロード失敗（404 / CORS / Range 非対応で全ダウンロードになる等）→ 画面にエラーメッセージ、コンソールに詳細を出す。
- `page` が範囲外 → `1`〜最終ページにクランプする。
- 不正なパラメタ（`binding` / `spread` が未知の値）→ 既定値にフォールバックする。

## 9. テスト方針

- **`navigation.js` の純ロジックを単体テストの中心**とする:
  - 見開きペアの組み方（`1-2 / 3-4 …`）。
  - 綴じ方向による左右の並び（`rtl` / `ltr`）。
  - `page` から表示ペアの算出。
  - スライダー値 ↔ ページの対応、移動の刻み。
  - 範囲外ページのクランプ、末尾奇数ページの単独表示。
- 描画系（`pdf-core` / `viewer`）は実 PDF を用いた手動確認とする。

## 10. デプロイ / 動作確認

- nginx で `tiny-pdf-viewer/` 一式と PDF 置き場を静的配信する。
- Range Request の確認:

  ```sh
  curl -I -H "Range: bytes=0-1023" https://example/pdfs/big.pdf
  # → 206 Partial Content と Content-Range が返れば OK
  ```

- `application/pdf` を gzip 対象に含めないこと。
